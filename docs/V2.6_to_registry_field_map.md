# V2.6 → Registry Field Map (Sprint 1)

*Created: 2026-04-26 by Claude (Cowork session)*
*Purpose: single source of truth for translating Environment–Cognition Taxonomy V2.6 columns into JSON paths in `core/trs-core/v0.2.8/registry/registry_v0.2.8.json`. Every Phase B/C author must consult this document before writing or editing a tag entry.*

---

## 1. Scope

This map covers the eighteen social-interaction latents L41–L58. The same map will, with minor extensions, govern Sprint 2 (cognitive/affect latents L01–L40) once a parallel construct review is completed for those.

The map reflects the registry schema as it stands on `track1/latents/L41-L58-claude-sprint1`, which extends the v0.2.8 schema with five new fields adopted by the architectural panel (28 April 2026, see `docs/sprint1_panel_disposition_2026-04-28.md`).

---

## 2. The five panel-added fields

These fields are new in this sprint. They do not appear in the December 2025 release of v0.2.8, but they are necessary to honour the panel disposition. A schema-version bump (v0.2.8 → v0.2.9) will be cut at the Phase G release-candidate.

| New field | Type | Required? | Default | Source |
|---|---|---|---|---|
| `extractability.requires_floor_plan` | boolean | yes | `false` | Hillier (panel) |
| `extractability.configurational_measure` | enum: `integration` ∣ `depth` ∣ `connectivity` ∣ `isovist_area` ∣ `none` | yes | `none` | Hillier (panel) |
| `secondary_value_type` | enum: same set as `value_type` | optional | (omitted) | Kendon (panel) — used when primary value is categorical and secondary is the affordance-strength Likert |
| `notes.interaction_mode` | enum: `focused` ∣ `unfocused` ∣ `mixed` | yes | (must be set explicitly) | Goffman (panel) |
| `notes.cross_cultural_variance` | enum: `low` ∣ `medium` ∣ `high` | yes | `medium` | Hall (panel) |

The five additions are minimal in surface area but consequential in semantics. They are documented here so that the JSON-schema validators in `core/trs-core/v0.2.8/schemas/` can be extended in lockstep during Phase D.

## 3. Field-by-field map

The V2.6 spreadsheet has six columns of interest plus a workbook sheet (`Latent_Indicators`) referenced by ID. Mapping:

| V2.6 column | → registry JSON path | Authoring note |
|---|---|---|
| `ID` (e.g., `L44`) | `attribute_id` (verbatim, e.g., `"L44"`) AND mention in `notes.l_number` for redundancy | Do *not* use the L## as the canonical name |
| `Latent` (e.g., "Sociopetal Seating Topology") | `definition_short` (verbatim, ≤ 90 chars) | Consider this the human-readable handle |
| `Definition` (e.g., "Seating geometry supporting face-to-face talk (F-formations)") | `semantics.definition_long` (expanded to ≥ 200 chars with primary literature citation inline) | The V2.6 short definition seeds the long one but is not sufficient |
| `Observable 2D Cues` | `semantics.extraction_notes_2d` (verbatim list as a stem; expanded with operationalisation against available upstream observables) | Use as a starting point — replace cue-language with available-upstream-observable language per Phase B Engineer pass |
| `Tier` (1, 2, or 3) | governs `extractability.from_2d` / `from_3d_vr` / `monocular_3d_approx` per the table in §4 below | Override of V2.6 Tier requires explicit justification in `notes.extraction_notes` |
| `Latent_Indicators` workbook sheet (3–6 rating items + 1–2 behavioral proxies) | `semantics.examples_positive` (≥ 6 from rating items) AND `semantics.examples_negative` (≥ 4 from contrasts) AND `semantics.factor_associations` (the indicator structure itself) | Behavioral proxies are not registry-level content — they belong in extractor docstrings |

## 4. Tier → extractability mapping

This is the unmodified V2.6 mapping for fifteen of the eighteen latents; the three exceptions are noted. **Note:** the registry's `extractability` fields are enum-typed `{"yes", "partial", "no"}`, not boolean — `validate_registry.py` enforces this.

| V2.6 Tier | `from_2d` | `from_3d_vr` | `monocular_3d_approx` | Latents at this tier |
|---|---|---|---|---|
| 1 | `"yes"` | `"yes"` | `"yes"` | L42, L43, L44, L45, L46, L48, L49, L50, L51, L52, L53, L56 |
| 2 | `"partial"` | `"yes"` | `"partial"` | L47, L54, L55, L58 |
| 3 | `"no"` | `"no"` | `"no"` | L57, **L41 (panel-overridden from V2.6 Tier 2)** |

The override for L41 is documented on Hillier's argument that chance-encounter potential is a configurational property of the whole spatial system and therefore cannot be inferred from a single image regardless of richness. See panel disposition §1.

For latents with `extractability.requires_floor_plan: true`, the `from_2d` flag is set per the Tier above but the registry consumer is expected to combine the image with floor-plan augmentation; this is a downstream contract, not a registry-level constraint.

## 5. value_type assignment

| value_type | Latents | value_range | unit |
|---|---|---|---|
| `categorical` (with `secondary_value_type: ordinal`) | L44, L48, L49 | enum: `[vis_a_vis, l_arrangement, side_by_side, semicircular, circular, none]`; secondary range `[0, 4]` | `enum`; secondary `likert_0_4` |
| `latent_score` (binary) | L46, L52, L53 | `[0, 1]` | `binary` |
| `ordinal` (Likert 5-point) | L41, L42, L43, L45, L47, L50, L51, L54, L55, L56, L57, L58 | `[0, 4]` | `likert_0_4` |

The Kendon-driven categorical-with-secondary-Likert pattern is new with Sprint 1. The schema extension required to support it is a single boolean check in `scripts/validate_registry.py`: if `value_type == "categorical"` and `secondary_value_type` is present, validate `secondary_value_type` against the same enum as `value_type`.

## 10. Authoring checklist (every Phase C entry)

Before merging any L## entry into `registry_v0.2.8.json`, verify that:

1. The thirteen `Sprint 1 architectural decision` fields above are populated according to this map
2. The five panel-added fields are populated
3. `extraction.compute_from` does not share more than 50% of its members with any other latent in L41–L58 (Spohn rule, enforced by `audit-extraction-plan` in Phase D)
4. `semantics.examples_positive` ≥ 6 and `semantics.examples_negative` ≥ 4
5. `semantics.scope_includes` ≥ 3 and `semantics.scope_excludes` ≥ 3
6. `literature.key_refs` ≥ 1 with full APA formatting and a real DOI where applicable
7. The V2.6 Tier override for L41 is documented in `notes.extraction_notes`
8. The Sprint-2 ranking-theoretic flag is set in `notes.bn_notes` for L46, L52, L53
9. The blind-read challenge result is recorded in `notes.validation_notes`
